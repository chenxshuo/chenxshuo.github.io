两天入门kaggle-Titanic

前言

在学习了一部分pandas库的使用之后，便可以通过参加比赛，使用一些实战数据集来练习和检验学习的成果，并加深理解和实战技巧。这次参加了Kaggle的入门比赛Titanic: Machine Learning from Disaster,根据一些博客的指导，尝试了两次，提交了三次结果，准确率从61%提升到79.9%，排名为1526(top 18%) ，虽然成绩依然十分辣鸡，但是在这第一次实战中收获了许多。

比赛简介

Titanic: Machine Learning from Disaster，是Kaggle网站上的一个入门比赛，数据集给出泰坦尼克号上乘客的相关信息，参赛者通过数据集训练模型来预测乘客是否幸存。

相关属性与含义如下表：

Variable Definition Key
survival Survival 0 = No, 1 = Yes
pclass Ticket class 1 = 1st, 2 = 2nd, 3 = 3rd
sex Sex
Age Age in years
sibsp # of siblings / spouses aboard the Titanic
parch # of parents / children aboard the Titanic
ticket Ticket number
fare Passenger fare
cabin Cabin number
embarked Port of Embarkation C = Cherbourg, Q = Queenstown, S = Southampton

第一次尝试

第一次尝试十分简单和草率，直接删去了大量数据，简单数据清洗之后，便套用模型得出结果，最终得到了61%的正确率，也就是比瞎猜好了一点，效果比较差。

观察属性和是否幸存的数量关系

通过观察一些属性对应的幸存人数，可以比较直观和简单的看出每个属性对于结果的影响程度

去除“无效”属性

在第一次的尝试中，我直接去除了比较多的看似没有什么用的特征，比如姓名，ID，票的号码，登船的地点，以及船舱号等

Fare Column

在之前的预览中可以看到Fare这个属性存储的是float数据，并且在test数据集中有一个缺失值。

我们在这里使用平均值来填充空缺值

Age Column

在预览中可以看到Age属性在两份数据集中都有大量的缺失值(分别是177个和86个),对于这种有大量缺失值并且明显比较重要的属性，在第一次尝试中，使用了不是很合理的随机数填充；在第二次填充中，使用了根据与其他属性的联系来确定缺失值的方法，下文写到时再详细阐述。

填充随机数的取值范围为mean-std~mean+std,即平均数上下各一个标准差的范围，随机数的个数为缺失值的数量

Family Column

第一次尝试中简单对待了这个特征，仅仅将其转化为了是否有家庭成员这个二维特征而并未关注其家庭成员的数量。

Ticket Class Column

从预览中可以看到Pclass属性有1,2,3三个值，并且与是否幸存有比较明显的关系。

在第一次尝试中并未对该数据进行处理

Sex Column

性别有男女两个值，该属性的值为字符串female和male，可以将其分别赋值为1和0

Step.3 特征工程

组合数据集

这是一个比较常用的方法，把训练数据集和测试数据集组合一起可以帮助我们更好的提取特征的信息，而避免把在测试数据集中的重要特征忽略掉。

Titles Column

特征工程的常用的方法是从已有的特征中提取新的特征，比如在这个数据集中有一个特征为Name，每个人的Name都有一个身份标示比如Mr, Mrs, Miss,Sir, Dr等，这些称谓反映了其身份地位的高低，而身份地位的高低在一定程度上会影响到生存的可能性大小，因此可以把这个Title的特征提取出来。

具体的实现为先把属性Name中的表示地位的称呼提取出来，再统一转换为表示地位的词语。

在原数据中，Name的格式为

Braund, Mr. Owen Harris

Futrelle, Mrs. Jacques Heath (Lily May Peel)

得到称谓的方法是先去除’,’再去除’.’

Age Column

对于Age这个属性，原数据中有大量的空缺值，在第一次尝试中，随机在$average+-std$ 的范围内填充数值。在这次尝试中，通过参考其他相关属性的关联关系来更好的确定Age. 例如我们通过数据处理可以大致估计Pclass为1，并且称谓为Miss的乘客的年龄大致为30，这样填充空缺值的可信度和训练的准确度会有所提高。

之后就可以看到输出结果

此时我们就可以通过Sex，Pclass和Title来比较精确的推断所空缺的年龄值

Name Column

姓名这一个属性我们把Title提取出来之后就可以去除掉,除此之外，对于Title有一个常用的方法为dummy编码，因为Title的值均为代表不同类别的字符串，无法直接进行数学处理，那么可以把每个类别当做一个新并且属性取值均为0或者1，这就是dummy编码所做的事。

Fare Column

这个属性有一个空缺值，直接使用中位数填充即可

Embarked Column

使用S填充空缺值，之后同样使用dummy编码

Cabin Column

Cabin这个属性有更多的空缺值(1014/1309)，秉着尽量不要舍弃特征的前提，可以为空缺值赋予U，替换其他值为对应的首字母比如C2014换为C，之后使用dummy编码。

Sex Column

Sex这个属性有两个值一个是female,一个是male，可以转换为0和1

Pclass(Ticket Class ) Column

Pclass有1,2,3等值，避免数值的大小带来并不需要的影响，可以使用dummy编码转换一下

Ticket Column

Ticket这个属性代表船票号码，在这里简单处理了一下，把全是数字的号码使用XXX代替，其余开头有字母的用前三个字符代替(觉得这个属性还可以提取出更多的信息以提高准确度)

Family Column

Family这个特征基于SibSp和Parch两个特征得到，常理来讲家庭成员越多，那么可以互相帮助，最终幸存的概率要大一点，单独乘坐的话幸存几率要小一点。那么可以把两个值加起来再加上1即自己，之后定义Single，SmallFamily， LargeFamily三个特征。
